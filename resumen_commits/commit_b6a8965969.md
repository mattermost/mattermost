### Descripción de los Cambios
Implementa la funcionalidad solicitada en GitHub issue #19916 para listar únicamente usuarios desactivados/inactivos en mmctl.

Cambios principales:
1. **Nuevo flag `--inactive`**: Agrega la opción `--inactive` al comando `mmctl user list` para filtrar solo usuarios desactivados (donde `DeletedAt != 0`)
2. **Nueva interfaz de cliente**: Expone el método `GetUsersWithCustomQueryParameters` en la interfaz del cliente mmctl que permite pasar parámetros de consulta URL personalizados
3. **Refactorización**: Simplifica la lógica de listado de usuarios para usar parámetros de consulta en lugar de múltiples llamadas a diferentes métodos API (`GetUsers` vs `GetUsersInTeam`)
4. **Tests actualizados**: Agrega tests exhaustivos para los nuevos casos de uso incluyendo usuarios inactivos, paginación con filtro de inactivos, y combinación con filtro de equipo
5. **Documentación**: Actualiza la documentación de mmctl para reflejar la nueva opción

### Veredicto del Cambio
A) Cambio útil que mejora/corrige - Implementa funcionalidad solicitada por la comunidad para gestión de usuarios
