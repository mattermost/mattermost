### Descripción de los Cambios
Este commit corrige la acción de copiar texto en el menú de opciones de mensaje (dot menu) para usar el mensaje original sin procesar.

**Cambio realizado:**
En el archivo `dot_menu.tsx`, la función `copyText` ahora utiliza `message_source` en lugar de `message`:

```typescript
// Antes:
Utils.copyToClipboard(this.props.post.message);

// Después:
Utils.copyToClipboard(this.props.post.message_source || this.props.post.message);
```

**Explicación:**
- `message`: Contiene el mensaje procesado/prenderizado (puede incluir formato Markdown convertido, emojis procesados, etc.)
- `message_source`: Contiene el texto original del mensaje tal como lo escribió el usuario

Este cambio asegura que cuando un usuario copia el texto de un mensaje, obtenga el contenido original sin formato en lugar de la versión procesada, lo cual es más útil para pegar en otros lugares.

El operador `||` garantiza compatibilidad hacia atrás: si `message_source` no está disponible, se usa `message` como fallback.
