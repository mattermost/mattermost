### Descripción de los Cambios
Este commit corrige la definición de tipos para constantes de eventos WebSocket, asegurando type safety en el código Go.

**Problema identificado:** Varias constantes de eventos WebSocket no tenían el tipo explícito `WebsocketEventType`, lo que podía causar problemas de type safety.

**Cambios realizados:**

1. **En `websocket_message.go`**: Se añade el tipo `WebsocketEventType` a las siguientes constantes que estaban sin tipo:
   - `WebsocketEventChannelBookmarkCreated`
   - `WebsocketEventChannelBookmarkUpdated`
   - `WebsocketEventChannelBookmarkDeleted`
   - `WebsocketEventChannelBookmarkSorted`
   - `WebsocketPostedNotifyAck`

2. **En `system.go`**: Se añade una conversión explícita a string para `model.WebsocketPostedNotifyAck` al registrar el handler del router WebSocket:
   ```go
   api.Router.Handle(string(model.WebsocketPostedNotifyAck), ...)
   ```

Estos cambios aseguran que todas las constantes de eventos WebSocket tengan el tipo correcto, previniendo posibles errores de tipo en tiempo de compilación y mejorando la claridad del código.
