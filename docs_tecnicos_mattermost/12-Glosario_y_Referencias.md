# 12 - Glosario y Referencias

## Glosario de Términos

### A

| Término | Definición |
|---------|------------|
| **API** | Application Programming Interface - Conjunto de endpoints para interactuar con Mattermost |
| **App Layer** | Capa de aplicación que contiene la lógica de negocio del servidor |
| **Audit Log** | Registro de acciones realizadas en el sistema para fines de auditoría |

### B

| Término | Definición |
|---------|------------|
| **Backend** | Parte del servidor desarrollada en Go que maneja la lógica y datos |
| **Bot** | Cuenta automatizada que puede enviar mensajes y realizar acciones mediante API |
| **Broadcast** | Envío de un mensaje WebSocket a múltiples clientes simultáneamente |

### C

| Término | Definición |
|---------|------------|
| **Cache Layer** | Capa de caché del store que mejora el rendimiento de consultas frecuentes |
| **Channel** | Canal de comunicación dentro de un equipo (público, privado, directo o grupal) |
| **Cluster** | Configuración de múltiples servidores Mattermost para alta disponibilidad |
| **CSRF** | Cross-Site Request Forgery - Ataque mitigado mediante tokens en formularios |

### D

| Término | Definición |
|---------|------------|
| **DM** | Direct Message - Mensaje directo entre dos usuarios |
| **Docker** | Plataforma de contenedores utilizada para desarrollo y despliegue |

### E

| Término | Definición |
|---------|------------|
| **eDiscovery** | Proceso de descubrimiento electrónico para cumplimiento legal |
| **Elasticsearch** | Motor de búsqueda utilizado para indexación de contenido |
| **Enterprise** | Versión de pago de Mattermost con características avanzadas |
| **E2E** | End-to-End - Pruebas que simulan interacciones de usuario completas |

### F

| Término | Definición |
|---------|------------|
| **FileInfo** | Estructura de datos que representa un archivo subido al sistema |
| **Frontend** | Aplicación web desarrollada en React/TypeScript |

### G

| Término | Definición |
|---------|------------|
| **GM** | Group Message - Mensaje grupal entre 3-8 usuarios |
| **Go** | Lenguaje de programación utilizado para el backend |
| **Gorilla Mux** | Router HTTP utilizado en la API REST |

### H

| Término | Definición |
|---------|------------|
| **HA** | High Availability - Configuración de alta disponibilidad |
| **Hook** | Punto de extensión donde los plugins pueden interceptar eventos |
| **Hub** | Componente central que gestiona conexiones WebSocket |

### I

| Término | Definición |
|---------|------------|
| **i18n** | Internationalization - Soporte para múltiples idiomas |
| **IPC** | Inter-Process Communication - Comunicación entre proceso principal y plugins |

### J

| Término | Definición |
|---------|------------|
| **Job** | Tarea programada que se ejecuta en segundo plano |

### K

| Término | Definición |
|---------|------------|
| **KV Store** | Key-Value Store - Almacenamiento para plugins basado en clave-valor |

### L

| Término | Definición |
|---------|------------|
| **LDAP** | Lightweight Directory Access Protocol - Protocolo para autenticación de directorios |
| **Local Cache** | Caché en memoria LRU para datos frecuentemente accedidos |

### M

| Término | Definición |
|---------|------------|
| **MFA** | Multi-Factor Authentication - Autenticación de múltiples factores |
| **Migration** | Script de cambio en el esquema de base de datos |
| **Model** | Estructura de datos que representa una entidad del sistema |
| **Monorepo** | Repositorio único que contiene múltiples proyectos relacionados |

### N

| Término | Definición |
|---------|------------|
| **Notification** | Alerta enviada al usuario sobre actividad relevante |

### O

| Término | Definición |
|---------|------------|
| **OAuth** | Protocolo de autorización para acceso delegado |
| **OpenAPI** | Especificación estándar para APIs REST (Swagger) |

### P

| Término | Definición |
|---------|------------|
| **Permission** | Acción específica que un usuario puede realizar |
| **Plugin** | Extensión que añade funcionalidad al sistema |
| **Post** | Mensaje enviado en un canal |
| **Props** | Propiedades adicionales almacenadas como JSON |

### R

| Término | Definición |
|---------|------------|
| **RBAC** | Role-Based Access Control - Control de acceso basado en roles |
| **React** | Biblioteca de JavaScript utilizada para el frontend |
| **Redux** | Biblioteca de gestión de estado del frontend |
| **Role** | Conjunto de permisos asignado a usuarios |
| **RPC** | Remote Procedure Call - Llamada a procedimiento remoto |

### S

| Término | Definición |
|---------|------------|
| **SAML** | Security Assertion Markup Language - Protocolo para autenticación single sign-on |
| **Scheme** | Conjunto de roles predeterminados para equipos o canales |
| **Selector** | Función que extrae datos del estado de Redux |
| **Session** | Período de autenticación de un usuario |
| **Slash Command** | Comando invocado con `/comando` en el chat |
| **SQLx** | Extensión de database/sql para Go |
| **Store** | Capa de abstracción para acceso a datos |

### T

| Término | Definición |
|---------|------------|
| **Team** | Equipo de trabajo que contiene canales y usuarios |
| **Thread** | Conversación en hilo (respuestas a un post) |
| **TLS** | Transport Layer Security - Protocolo de cifrado |
| **TOTP** | Time-based One-Time Password - Contraseña de un solo uso basada en tiempo |
| **Thunk** | Función que permite lógica asíncrona en acciones Redux |

### U

| Término | Definición |
|---------|------------|
| **User** | Cuenta de usuario en el sistema |

### W

| Término | Definición |
|---------|------------|
| **Webhook** | Callback HTTP que notifica eventos a sistemas externos |
| **WebSocket** | Protocolo de comunicación bidireccional en tiempo real |
| **Workspace** | Entorno de trabajo en el monorepo npm |
