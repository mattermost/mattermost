# Mattermost Development Guide

## Cursor Cloud specific instructions

### Architecture Overview

Mattermost is a monorepo with two main components:
- **Server** (`server/`): Go backend serving REST API + WebSocket on port 8065
- **Webapp** (`webapp/`): React/TypeScript frontend using npm workspaces

### Required Runtime Versions

- **Go**: Version in `server/.go-version` (currently 1.25.8)
- **Node.js**: Version in `.nvmrc` (currently 24.11), requires npm 11.x
- **Docker**: Required for PostgreSQL (the only truly required service)

### Starting Docker Services

PostgreSQL is the only required Docker service. Start it with:
```
cd server && ENABLED_DOCKER_SERVICES="postgres" make start-docker
```
Default PostgreSQL credentials: `mmuser/mostest`, database: `mattermost_test`, port: `5432`.

To start all default dev services (postgres, inbucket, redis, prometheus, grafana, loki, otel-collector, minio):
```
cd server && make start-docker
```

### Building and Running

**Server** (from `server/`):
```
make run-server        # Full run: builds server + starts Docker services
go run -buildvcs=false ./cmd/mattermost   # Direct run (if Docker already started)
```

**Webapp** (from `webapp/`):
```
npm run build          # Production build (output in webapp/channels/dist/)
npm run dev-server     # Dev server with hot reload
```

The server serves the webapp static files from `server/client` (a symlink to `webapp/channels/dist/`). Create it with:
```
cd server && ln -nfs ../webapp/channels/dist client
```

### Running Tests

**Server tests** (from `server/`):
```
go test -buildvcs=false ./public/model/          # Single package
go test -buildvcs=false -run TestName ./path/...  # Single test
make test-server                                  # Full test suite (needs Docker services)
```

**Webapp tests** (from `webapp/`):
```
npm run test                                          # All workspace tests
npm run test --workspace platform/client              # Specific workspace
cd channels && npx jest --testPathPatterns="path"     # Specific test file in channels
```

### Linting

**Server**: `cd server && go vet -buildvcs=false ./...` (for full lint: `make golangci-lint`)
**Webapp**: `cd webapp && npm run check` (runs ESLint + Stylelint across all workspaces)

### Key Gotchas

- Always pass `-buildvcs=false` to Go build/test commands (set via `GOFLAGS` in Makefile).
- The `go.work` file is auto-generated by `make setup-go-work`. Run it if Go module resolution fails. Pass `BUILD_ENTERPRISE=false` if the enterprise directory is not present.
- The SMTP connection error on server startup ("Mail server connection test failed") is expected when inbucket is not running — it does not prevent the server from functioning.
- Jest 30.x uses `--testPathPatterns` (not `--testPathPattern`).
- Webapp npm `postinstall` builds platform types/client/components/shared — if `npm install` fails, check those workspace builds.
- The server auto-generates a default config on first run when `config/config.json` doesn't exist.
